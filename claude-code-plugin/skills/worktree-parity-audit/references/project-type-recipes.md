## Node / TypeScript

**Typically gitignored**:
- `node_modules/` ‚Äî the big one
- `.next/`, `.nuxt/`, `.svelte-kit/`, `dist/`, `build/` ‚Äî build outputs, regenerated by golden command
- `.env*` except `.env.example`
- `coverage/`, `.turbo/`, `.cache/` ‚Äî caches
- `*.tsbuildinfo` ‚Äî incremental TS build state

**Standard install**:

| Package manager | Command | Lockfile |
|---|---|---|
| pnpm | `pnpm install --frozen-lockfile` | `pnpm-lock.yaml` |
| npm | `npm ci` | `package-lock.json` |
| yarn | `yarn install --frozen-lockfile` | `yarn.lock` |
| bun | `bun install --frozen-lockfile` | `bun.lockb` |

Always prefer the frozen variant in `worktree-setup-commands` ‚Äî it fails on lockfile drift instead of silently mutating it.

**Gotchas**:
- Monorepo workspaces (`pnpm-workspace.yaml`, `package.json#workspaces`) need install at the repo root, not per-package. Sortie's worktrees are at repo root, so this is fine.
- Native modules (`sharp`, `node-canvas`, `better-sqlite3`) ‚Äî fresh install per worktree can take 30s+. If same-FS and the project doesn't mutate `node_modules` at runtime, share via `worktree-sync-paths.link: [node_modules]` for instant setup.
- `.npmrc` / `.yarnrc.yml` ‚Äî usually committed; if gitignored (auth tokens), share via `link:`.
- Prisma: `pnpm prisma generate` after install if `prisma/schema.prisma` exists; `pnpm prisma migrate deploy` for DB.

## Python

**Typically gitignored**:
- `.venv/`, `venv/`, `env/` ‚Äî virtualenv
- `__pycache__/`, `*.pyc` ‚Äî bytecode
- `.pytest_cache/`, `.mypy_cache/`, `.ruff_cache/` ‚Äî caches
- `.env`, `.envrc`
- `*.egg-info/`

**Standard install**:

| Tool | Setup |
|---|---|
| uv | `uv sync --frozen` |
| Poetry | `poetry install --no-root` |
| pip + venv | `python -m venv .venv && .venv/bin/pip install -r requirements.txt` |
| pip + pyproject | `python -m venv .venv && .venv/bin/pip install -e .[dev]` |
| Pipenv | `pipenv install --dev` |

**Gotchas**:
- `uv` is by far the fastest ‚Äî sub-second on a warm cache. Recommend it if the project uses it.
- Shared `.venv/` via `link:` works for read-only deps but breaks when pytest writes `.pyc` into installed packages. Prefer per-worktree venv + a fast installer.
- C extensions (`numpy`, `psycopg2`, `cryptography`) may need system headers ‚Äî assume the dev environment has them; flag if compilation fails in the parity check.
- Activate handling: most package managers use absolute paths in `.venv/bin/activate`, which break when the venv is hard-linked across worktrees. Recreate, don't share.

## Go

**Typically gitignored**:
- `vendor/` (rare ‚Äî usually committed if used)
- `bin/`, `dist/` ‚Äî build outputs

**Standard install**:
- `go mod download` (optional ‚Äî `go test ./...` does it implicitly)
- `go generate ./...` if codegen is used

**Gotchas**:
- Module cache (`$GOMODCACHE`, default `$GOPATH/pkg/mod`) is global, shared across worktrees ‚Äî no setup needed.
- Build cache (`$GOCACHE`) is also global.
- If `vendor/` is committed, nothing to do. If gitignored but referenced (`-mod=vendor`), add `go mod vendor` to setup commands.
- `tools.go` pattern: tools listed in `go.mod` are fetched by `go mod download` but binaries are not built until `go install` or `go run` ‚Äî fine for tests but flag if a `Makefile` runs `protoc-gen-go` etc.

Go is the most parity-friendly stack out of the box.

## Rust

**Typically gitignored**:
- `target/` ‚Äî build output (large, slow to rebuild)
- `Cargo.lock` for libraries; committed for binaries

**Standard install**:
- `cargo fetch` (warms the registry cache; optional)
- The verify command (`cargo test`, `cargo clippy`) does the build itself

**Gotchas**:
- `target/` is per-worktree by default. Cold rebuild can take minutes. Two ways to speed it up:
  - Set `CARGO_TARGET_DIR` to a shared location (e.g., `~/.cargo-target-shared/<project>`). Recommended over sharing the directory itself. Configure via `worktree-setup-command` exporting the env var, or via a project-level `.cargo/config.toml`.
  - Share via `worktree-sync-paths.link: [target]` (same-FS only). Risk: lockfile contention if two worktrees `cargo build` simultaneously. `CARGO_TARGET_DIR` is safer.
- Registry cache (`$CARGO_HOME/registry`) is global ‚Äî no per-worktree setup.

## Ruby / Rails

**Typically gitignored**:
- `vendor/bundle/` (if `bundle config set --local path 'vendor/bundle'`)
- `tmp/`, `log/`
- `db/*.sqlite3` ‚Äî SQLite dev DB
- `config/master.key`, `config/credentials/*.key` ‚Äî sensitive
- `.env`, `.env.local`
- `node_modules/` (asset pipeline)

**Standard install**:
- `bundle install` (system gem path, no copy) or `bundle install --path vendor/bundle` (local)
- `bin/rails db:prepare` for DB
- `pnpm install` (or yarn/npm) if assets use a Node toolchain

**Gotchas**:
- `config/master.key` is required for Rails to decrypt credentials. Share via `worktree-sync-paths.link: [config/master.key]`.
- Per-worktree SQLite is correct; multiple worktrees on a shared DB will lock-contend.
- `bin/setup` (if present) is often the canonical recipe ‚Äî read it before guessing.

## Elixir / Phoenix

**Typically gitignored**:
- `deps/` ‚Äî dependencies
- `_build/` ‚Äî compiled output
- `.elixir_ls/` ‚Äî language server cache

**Standard install**:
- `mix deps.get`
- `mix compile`
- `mix ecto.create && mix ecto.migrate` if using Ecto

**Gotchas**:
- `_build/` is per-arch / per-env. Per-worktree is correct.
- Hex cache (`$MIX_HOME/.hex`) is global.

## JVM (Maven / Gradle)

**Typically gitignored**:
- `target/` (Maven), `build/` (Gradle), `.gradle/`
- `*.class`

**Standard install**:
- `./mvnw verify -DskipTests` or `./gradlew build -x test` to pre-populate.
- Test command (`./mvnw test`, `./gradlew test`) downloads deps lazily.

**Gotchas**:
- Caches at `~/.m2/repository` (Maven) and `~/.gradle/caches` (Gradle) are global ‚Äî no per-worktree setup.
- Wrapper scripts (`mvnw`, `gradlew`) are committed and executable.

## PHP (Composer)

**Typically gitignored**:
- `vendor/`
- `.env*`

**Standard install**:
- `composer install --no-interaction --prefer-dist` (dev)
- `composer install --no-dev` (prod)

**Gotchas**:
- Composer's global cache makes fresh `composer install` fast on a warm machine ‚Äî recreate over share.

## Swift

**Typically gitignored**:
- `.build/`, `Packages/`
- `*.xcodeproj/xcuserdata/`

**Standard install**:
- `swift package resolve`
- `swift build`

**Gotchas**:
- macOS-only stack. Sortie on Linux won't help; flag and stop.
