# Node / TypeScript — Worktree Parity Recipe

Marker files: `package.json`, `pnpm-workspace.yaml`.

## Typically gitignored

- `node_modules/` — the big one
- `.next/`, `.nuxt/`, `.svelte-kit/`, `dist/`, `build/` — build outputs, regenerated by the golden command
- `.env*` except `.env.example`
- `coverage/`, `.turbo/`, `.cache/` — caches
- `*.tsbuildinfo` — incremental TS build state

## Standard install

| Package manager | Command | Lockfile |
|---|---|---|
| pnpm | `pnpm install --frozen-lockfile` | `pnpm-lock.yaml` |
| npm | `npm ci` | `package-lock.json` |
| yarn | `yarn install --frozen-lockfile` | `yarn.lock` |
| bun | `bun install --frozen-lockfile` | `bun.lockb` |

Always prefer the frozen variant in `worktree-setup-commands` — it fails on lockfile drift instead of silently mutating it.

## Gotchas

- **Monorepo workspaces** (`pnpm-workspace.yaml`, `package.json#workspaces`) need install at the repo root, not per-package. Sortie's worktrees are at repo root, so this is fine.
- **Native modules** (`sharp`, `node-canvas`, `better-sqlite3`) — fresh install per worktree can take 30s+. If same-FS and the project doesn't mutate `node_modules` at runtime, share via `worktree-sync-paths.link: [node_modules]` for instant setup.
- **`.npmrc` / `.yarnrc.yml`** — usually committed; if gitignored (auth tokens), share via `link:`.
- **Prisma**: run `pnpm prisma generate` after install if `prisma/schema.prisma` exists; run `pnpm prisma migrate deploy` for DB.
